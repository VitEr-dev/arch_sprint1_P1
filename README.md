Задание 1
    Уровень 1 Ответ:
Задача разделить фронтенд на микрофронтенды, чтобы в будущем их могли одновременно развивать независимые команды.
Руководствуясь принципами DDD (в данном случае: Ограниченный контекст и Единый язык) предлагаю разделить frontend приложения mesto на следующие  микрофронтенды:
    UserProfile  // редактирование атрибутов профиля пользователя
    Auth_Regist  // авторизация и аутонтификация существующего пользователя, регистрация нового пользователя
    Cards        // (основной домен - загрузка и демонстрация пользователям фотоконтента пользователя) загрузка и удаление фото, редактивнование локации, управление реакцией пользователей: лайки и дизлайки 
    Host         // создаем в рамках выбранного фреймворка создания микрофронтендов - Webpack Module Federation

Выбор фреймворка Webpack Module Federation обусловлен следующими факторами: один JS - фреймворк (react); позволяет независимым приложениям использовать общий код во время выполнения; команды, разрабатывающие отдельные микрофронтенды, смогут получить доступ к коду друг друга до развертывания; функция lazy loading позволяет загружать фрагменты кода по требованию, что сокращает время первоначальной загрузки и помогает оптимизировать ресурсы; host и микрофронтенды могут использовать общие зависимости, функция shared dependencies позволяет эффективно управлять версиями, т.о. приложение загрузит одну версию библиотеки.

Описание новых структур проекта для каждого микрофронтенда:
/microfrontendsMF

        index.html
        index.css               // стили для микрофронтенда
        /components
            App.js              // перенесено из исходного приложения (после редактирования App.jsx, удалить)

        /context                // в папке содержится дополнительный компонент из исходного приложения

        /vendors                // в папке содержатся элементы стиля - шрифты

            /footer             // в папке содержатся стили отдельных элементов для компонента footer
